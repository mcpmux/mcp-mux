## Developer Certificate of Origin (DCO)

All contributions must be signed off to certify that you have the right to submit them. This is done by adding a `Signed-off-by` line to your commit messages:

```
Signed-off-by: Your Name <your.email@example.com>
```

You can do this automatically by using the `-s` flag when committing:

```bash
git commit -s -m "Your commit message"
```

## How to Contribute

### Reporting Issues

- Check existing issues first
- Include steps to reproduce
- Include your OS and app version

### Pull Requests

1. Fork the repository
2. Create a feature branch (`git checkout -b feature/my-feature`)
3. Make your changes
4. Ensure tests pass
5. Sign off your commits (`git commit -s`)
6. Push and create a Pull Request

### Code Style

- Run `pnpm lint` before submitting
- Follow existing code patterns
- Keep changes focused and minimal

## Questions?

Open a discussion or issue if you need help.
